# OCR 翻译校对工具 — 开发日志

## v1.0 版本总结（2025-06-12 ~ 2025-06-15）

### 技术栈
SwiftUI + Swift | macOS 14+ | Xcode 26.5 | 23 源文件 | 零警告零错误

### 核心流水线
截图/上传 → 框选选区 → OCR（Vision 本地，不依赖 API）→ 翻译 → 句子分析

### 图片框选系统
- 拖拽创建选区 + 8 手柄精调（4 角 + 4 边）+ 选区内部拖拽移动
- SF Symbol 对角光标 / 十字星 / 手掌拖动 / 边缘 resize 光标
- 缩放 100%~500%：滚轮 + 触控板捏合 + 按钮，连续累计
- 实测基帧 + 百分比坐标映射裁剪，选框与 OCR 识别范围完全一致
- 滚动事件只响应图片区域，设置面板/弹窗打开时不穿透

### AI 翻译 & 分析
- 多模型支持：DeepSeek / ChatGPT (OpenAI) / Gemini (Google)，设置中切换
- AIService 自动分发 OpenAI 兼容接口 / Gemini 原生接口
- **校对模式**：语法树形式展示句子成分，无语法/词汇注解
- **学习模式**：语法树 + 语法点（附例句）+ 词汇注解（N1~N2）
- 模式缓存 + 模式切换自动触发翻译
- 工具栏 linear 进度条

### 词汇查询
- 输入框查词 + Cmd+K 聚焦 + 查询按钮，支持折叠展开
- 浮动卡片展示：单词 / 读音 / 释义 / 词性 / JLPT / 例句 / 用法说明
- Session 内缓存

### TTS 语音朗读
- `AVSpeechSynthesizer` 本地引擎，零安装零网络，日文 Kyoka 语音
- 连贯文本模式下可播放/停止

### OCR 相关
- Vision 框架，ja/zh-Hans/zh-Hant/en 四语言，accurate 模式
- 识别结果分段 + 连贯两种显示模式

### UI
- 三栏布局：图片 | OCR 识别 | 翻译+分析
- 工具栏固定高度防跳动
- 按钮样式统一，提示文字单行截断
- API Key 用 UserDefaults 存储

### 已知边界
- 双指快速滑动偶有闪烁（根因：NSEvent ~120Hz vs SwiftUI ~60Hz，需 CADisplayLink 批处理）
- 缩放以图片中心为基准（非光标中心，Photoshop 风格待实现）

### 分支
- `main`：初始版本
- `dev`：多模型 + TTS + OCR 连贯模式
- `fix/crop-accuracy`：选框裁剪修复 + 语法树显示
